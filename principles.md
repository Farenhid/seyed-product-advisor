# Seyed's Foundational Principles

**Read this file first** — before any chapter. All 95 sessions build on these principles.

When advising, check the user's project and decisions against this list. Name violated and aligned principles explicitly in the **Principles Lens** section of every response.

---

## Purpose & Customer

| Principle | What it means |
|-----------|---------------|
| **Meaningful Authentic Purpose** | Purpose is real, specific, and survives the 10-year test — not a slogan |
| **10x better or easier** | Don't enter markets with marginal improvement; be radically better or radically simpler |
| **Product-Market-Fit** | Find fit before scaling; avoid the product death cycle |
| **Push vs Pull** | Healthy growth is pulled by demand, not pushed by marketing spend |
| **Customer obsession** | Every single customer matters — humans, not numbers |
| **Empathetic / Human-centered** | Understand persona, context, and problem before building |
| **Build what a few users love** | Depth over breadth early; love before scale |
| **Frictionless experience** | Remove friction at every touchpoint |

---

## Product Discipline

| Principle | What it means |
|-----------|---------------|
| **Unbundling** | Break apart bloated incumbents; find the wedge |
| **Subtraction / Less is more** | Improve by eliminating, not adding; fight the feature death spiral |
| **Address root cause, not effect** | Fix the real problem; correlation isn't causation |
| **Think outside the box / Reframe** | Look at the problem from an unexpected angle |
| **Impact assessment / Value-net** | Find inefficiencies in the value chain |
| **Non-consumers / Low-end segment** | Start where incumbents ignore; long-tail first |
| **Inspire the undecided** | Reach people who haven't chosen yet |
| **Day 0 test** | Would you start this today with what you know now? (anti sunk-cost) |

---

## Trust & Execution

| Principle | What it means |
|-----------|---------------|
| **Build trust** | Trust is the foundation of every relationship |
| **Transparency** | Say what you mean; document what you decided |
| **Underpromise, overdeliver** | Set the right expectations |
| **Social proof** | Testimonials, ratings, case studies — let others vouch |
| **Trial** | Let users experience value before committing |
| **Curation** | Quality over quantity in what you offer |
| **Avoid status quo bias** | Question "how it's always been done" |
| **OODA Loop** | In critical situations: observe, orient, decide, act — fast |

---

## Barriers to Remove (Empower Users)

Empower people by removing **one** barrier so they can do the job themselves:

- **Time** — save it
- **Money** — reduce cost
- **Skill** — lower the skill floor
- **Resource** — provide what's missing
- **Access** — open a closed door

---

## Growth, Network & Platform

| Principle | What it means |
|-----------|---------------|
| **Positive feedback loop / Flywheel** | Each turn makes the next easier; compound, don't linearize |
| **Network effect** | An action by one user helps others (implicit or explicit) |
| **Product-led growth** | The product itself drives acquisition and retention |
| **Growth products** | Build products whose use creates more use |
| **Come for the tool, stay for the network** | Single-player value first, then network lock-in |
| **Community for one side** | Reviews, Q&A, forums — users help users |
| **Co-creation / Prosumer** | Users become creators, not just consumers |
| **Leverage existing networks** | Ride networks that exist; watch for overlap threats |
| **Marketplace liquidity** | If building a marketplace, liquidity is the only thing |
| **AI flywheel / Data network effect** | Data moat vs feature; real-time nature matters |
| **Hooked journey** | Design the habit loop deliberately |
| **Toothbrush test** | High frequency of usage = durable product |
| **Own the customer relationship** | Don't rent your users from a platform |
| **Supply goes where demand is** | Pull supply with demand, not the reverse |

---

## Competition & Business Model

| Principle | What it means |
|-----------|---------------|
| **Competition is for losers** | Escape competition by solving a unique problem |
| **Don't shrink the pie, grow the pie** | Bring value to the game; complement, don't just compete |
| **Never compete on price** | Race to bottom destroys everyone |
| **Co-opetition** | Sometimes partner with competitors on shared infrastructure |
| **Biz model innovation >> Product innovation** | When competition gets tough, change the game |
| **Bits scale, atoms don't** | Software eats the world; value comes from software |
| **Proprietary growth channel** | Own your distribution; don't depend on one gatekeeper |
| **Stay in the middle of money flow** | Align price with usage; predictability |
| **Moat / Defensibility** | Build something hard to copy |
| **Double down on your strategic weapon** | Know your one unfair advantage |
| **Marketing/ops spend vs tech/talent spend** | Invest in what compounds |

## Biases & Fallacies to Avoid

| Bias | What it looks like |
|------|-------------------|
| **Status quo bias** | "Everyone does it this way" |
| **Sunk cost fallacy** | "We already invested so much" |
| **Outcome bias** | Judging a decision only by its result, not the reasoning |
| **Solution attachment** | Married to your approach before understanding the problem |
| **Feature creep** | Adding surface area when core isn't working |
| **Metric obsession** | Optimizing numbers instead of mission |
| **Progress theater** | Shipping visible things to feel productive while hard problems stay unsolved |

---

## Autonomy & Growth Pace

- **Autonomy / Choice** — give users real choice, not fake options
- **A happy customer returns, refers, and co-creates**
- **Exponential growth comes from solid growth; solid growth from slow growth**
